# DaisyPod iOS

Native iPhone app for DaisyPod. This is the supported frontend target.

## Requirements

- Xcode 26.5 or newer
- iOS 26 SDK/runtime
- XcodeGen

## Generate and Build

`DaisyPod.xcodeproj` is generated by XcodeGen and is intentionally ignored by git. Commit changes to `project.yml`, source files, tests, entitlements, and assets instead.

```bash
cd apps/ios
xcodegen generate
xcodebuild -project DaisyPod.xcodeproj -scheme DaisyPod -destination 'platform=iOS Simulator,name=iPhone 13 mini' build
```

Use XcodeBuildMCP from Codex for simulator build/run, screenshots, UI inspection, and logs when working interactively.

For physical-device validation, keep the development team as a local command override rather than committing a personal team id into `project.yml`:

```bash
DAISYPOD_DEVELOPMENT_TEAM=<apple-team-id> \
npm run ios:install:device -- <device-udid> <coredevice-id>
```

The install script builds into a per-run DerivedData directory and installs the `DaisyPod.app` produced by that build.

The currently known local device ids are:

- iPhone 13 mini: device UDID `00008110-001959E22178801E`, CoreDevice id `26975EF1-D3E3-53AE-8AC5-D581B8AADC89`
- iPad mini: device UDID `00008130-00066DAC1091401C`, CoreDevice id `61B3A04C-C6FD-59F2-AF99-7513B2F169AB`

CloudKit does not show a normal in-app permission prompt. `CKContainer.accountStatus` reports available only when the device is signed into iCloud, iCloud Drive/CloudKit are allowed for the account, the app is signed by an Apple Developer team that has the `com.elephanthand.daisypod` App ID with iCloud/CloudKit enabled, and the `iCloud.com.elephanthand.daisypod` container exists for that team. If physical builds keep showing "iCloud sync not active", verify the team override, provisioning profile, and CloudKit container in Apple Developer before debugging app UI.

If Xcode reports `Personal development teams ... do not support the iCloud capability` or the Apple Developer website says the Developer Program membership has expired, renew/reactivate the paid membership first. After renewal, accept any pending Apple Developer agreements, refresh or re-add the Apple ID in Xcode Settings > Accounts, then retry the physical-device install. Free/personal teams cannot provision the DaisyPod CloudKit entitlement.

## Architecture

- SwiftUI app shell with native top navigation and a persistent player surface.
- SQLite-backed local object store with explicit schema versioning.
- Local-first repository APIs for library, Inbox, Queue, downloads, settings, playback state, sync actions, and tombstones.
- `CloudKitPersonalSyncEngine` for deterministic private-database record snapshots, live CloudKit private-zone upload/download when iCloud is available, and portable local restore after local/remote merge.
- Native OPML and JSON backup import/export using iOS document flows.
- Native `AVPlayer` audio engine with background audio, Now Playing metadata, and remote commands.
- Existing DaisyPod backend remains the server boundary for RSS proxying, PodcastIndex, public clips, YouTube import, silence maps, Smart Skip, health, and capabilities.
- Target personal sync is iCloud/CloudKit private-database sync. The server `/api/sync` path is no longer part of the native product runtime. `project.yml` declares the CloudKit entitlement for `iCloud.com.elephanthand.daisypod`, and the live Apple Account status provider checks that explicit container with graceful local-first fallback when iCloud or provisioning is not ready.
- Native server features are gated in the app by backend `/api/capabilities`, native service headers, and Sign in with Apple backend sessions. PodcastIndex search works with native headers so normal device installs can discover shows. Protected processing and publishing routes require signing in with Apple once; the backend stores an account-linked session and the app keeps its session token in Keychain. Public abuse-sensitive deployments may still add App Attest on top of Apple Sign In.

## Current Native Slice

Implemented now:

- Buildable native app scaffold.
- Local seed library and settings.
- Inbox, Library, History, Downloads, Search/Add, and Settings top-level surfaces.
- Native Settings appearance picker with Light as the baseline look, plus Dark and Vaporwave themes for visual styling, effects, and motion only.
- Native Settings controls for persisted playback speed, skip forward/back, resume rewind, autoplay, native audio preference, auto-download, Inbox downloads, delete-after-listen, Wi-Fi-only downloads, storage cap, Inbox sort order, and automatic feed refresh interval.
- Native Settings profile stats for time listened, podcast time heard, speed savings, silence savings, and top listened podcasts recorded from local `AVPlayer` playback.
- Episode and podcast detail navigation.
- Native Library filtering across podcast title, author, tags, description, feed URL, website URL, source URL, and external id.
- Native podcast detail management with show summary metadata, Library membership, Subscribe/Unsubscribe, per-show new-episodes-to-Inbox, per-show playback/server-intelligence overrides, manual feed refresh with per-show cooldown, played/unplayed filtering, persisted newest/oldest show sorting, send-all-unplayed-to-Inbox, and mark-all played/unplayed actions.
- Native artwork rendering in episode rows, Library rows, and PodcastIndex result rows, with cached artwork preferred for downloaded episodes.
- Native episode detail actions for Play Now, Play Next, Add to End, Send to Inbox, favorite/unfavorite, mark played/unplayed, download/delete download, clip publishing, server intelligence, and YouTube preparation.
- Native chapter list rendering for episodes with chapter metadata, including chapter seek buttons, active-current styling, and local progress persistence after jumps.
- Swipe actions for play, queue, archive/remove, and played state.
- Drag reorder for Queue, persisted through local episode state and sync actions.
- Bottom native player bar backed by `AVPlayer`, including skip controls, play/pause, a device-local runtime sleep timer, playback progress/stat telemetry, startup cueing for the first queued episode, and an expanded player/queue sheet with play, favorite/unfavorite, reorder, send-to-Inbox, remove, and played-state actions.
- Manual episode downloads into app-container storage, delete-download controls, and local-file playback through `AVPlayer`.
- Foreground download maintenance after launch, import, queue, triage, and played-state changes:
  queued episodes download first when auto-download is enabled, optional Inbox downloads follow, episode artwork is prefetched into device-local app storage, Wi-Fi-only remote requests disable expensive/constrained network access, inactive downloads are deleted after listen, and storage-cap pruning preserves favorites, Queue, then Inbox.
- Native foreground automatic feed refresh for due library shows using the configured refresh interval, plus native background refresh and processing task registration for download maintenance. Background tasks reuse the same local-first maintenance path as foreground launch/triage changes, and physical-device validation remains required because iOS scheduling is system controlled.
- App Intents and App Shortcuts for opening a main section, pre-filling Add with a podcast or YouTube URL, toggling playback, and preparing iCloud sync. Intent handoff is stored locally and consumed when the app becomes active.
- Native URL-scheme routing for `daisypod://add?url=...`, `daisypod://open?section=...`, direct section links such as `daisypod://queue`, playback toggle links, and sync links.
- Backend health, capabilities, Sign in with Apple backend sessions, PodcastIndex search, YouTube import/enrich/extract, Apple Account/iCloud status for personal sync, and CloudKit personal-sync record preparation/merge/upload.
- Native capability gates from `/api/capabilities` for PodcastIndex, YouTube import/enrich/extract, clip publishing, silence maps, and Smart Skip. Explicitly disabled backend features show local disabled-state text instead of making failing route calls; local-first behavior remains available without a server.
- Native Settings backend/data controls show pending local changes, prepared iCloud changes, and local snapshot counts so users can understand what would sync without exposing device-local download paths.
- RSS import through `/api/rss/parse`, including inline Podcasting 2.0 and Podlove chapter tags plus backend-resolved external Podcasting 2.0 chapter JSON. Direct native RSS/Atom parsing remains available as an offline fallback and skips external chapter URLs.
- PodcastIndex search through `/api/podcast-index/search`, with Subscribe importing the selected RSS feed when the configured server advertises `podcastIndex.enabled`.
- YouTube video, playlist, channel, and podcast URL import through `/api/youtube/import` when the configured server advertises `youtubeImport.enabled`.
- YouTube episode detail controls for metadata enrichment through `/api/youtube/episodes/:id/enrich` and server-side audio preparation through `/api/youtube/episodes/:id/extract`.
- Episode detail clip composer with local clip persistence and optional public publishing through the existing `/api/clips` backend route.
- Episode detail server-intelligence controls for silence-map and Smart Skip requests, status checks, device-local SQLite cache display, category toggles for sponsors/ads, silence, self-promo, intros, and outros, and playback-time AVPlayer jumps when ready cached maps match enabled settings.
- OPML import/export and JSON backup/restore. OPML import uses the same RSS importer path as Add/refresh so it works through the configured backend when available and remains testable/local-first without a live server. Backup export strips device-local download paths; restore preserves the receiving device id, server URL, last sync timestamp, and existing local download fields.
- Unit and UI test targets.
